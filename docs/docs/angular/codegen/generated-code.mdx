```typescript

  public constructor(variables?: BooksQueryVariables, context?: Context) {
```

```typescript
// OR

```

## Mutation
Secondly, we have `AddBookMutation` class:

This class is generated for `AddBook` GraphQL mutation and accepts a **mandatory** mutation variables constructor parameter and can be used to execute a mutation as follows:

```typescript
  this.apollo.mutate(new AddBookMutation({ book })).subscribe();

```

An instance of `AddBookMutation` has the following properties:

`AddBookMutationInfo` is a helper type alias generated for `MutationInfo<AddBookMutationData, AddBookMutationVariables>` which is used by `@MutationUpdate` decorator in a `@State` class.

:::info
Note that `AddBookMutation` class constructor accepts a **mandatory** variables parameter as opposed to `BooksQuery` which accepts an optional parameter.  
This is automatically determined by codegen when variables have a mandatory field, in order to provide compile time safety, ensuring that variables are always provided when required by the GraphQL schema.
:::
